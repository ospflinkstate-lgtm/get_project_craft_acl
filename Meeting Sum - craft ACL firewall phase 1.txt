
งานแสดงและนำข้อมูล flow traffic ของ Openobserve เพื่อที่จะนำมาทำเป็น craft ACL firewall phase 1

===========================================================================
วัตถุประสงค์: เพื่อรายงานความคืบหน้าการใช้ข้อมูล Traffic Flow จากระบบ OpenObserve 
มาวิเคราะห์และออกแบบนโยบายความปลอดภัย  สำหรับเครือข่ายเซิร์ฟเวอร์วง 10.27.101.xx

1. สรุปภาพรวมงาน
- สร้าง html แสดงข้อมูลดิบ (Raw JSON) และให้แสดง  draft ACL Firewall for 101
- มีการดึงข้อมูล raw telemetry (JSON) จาก OpenObserve มาประมวลผลร่วมกับฐานข้อมูล csv เพื่อระบุตัวตนของ Host และ Subnet ได้เรียบร้อย
เปลี่ยนจากการคาดเดา เป็นการสร้างกฎบนพื้นฐานของ Traffic ที่เกิดขึ้นจริง (Evidence-based)


2. รายละเอียดการแสดงผลและออกแบบข้อมูล
การทดสอบสร้าง html เพื่อการนำข้อมูลมาแสดงผลโดย แนวคิดให้
- มีการยุบรวมหลาย Source และหลาย Service เข้าเป็นชุด Object Group เดียวกัน ช่วยลดจำนวนบรรทัด Config และลดความซับซ้อนในการดูแลรักษา
- แสดงเตือน Flow ที่มีปริมาณ Traffic สูงผิดปกติเพื่อเฝ้าระวังเป็นพิเศษ
- ให้สามารถยุบรวม IP รายเครื่องให้เป็นวงเครือข่าย (CIDR /24) เดียวกันได้อัตโนมัติหากพบพฤติกรรมกลุ่ม
- ให้ระบบสามารถค้นหา ได้หลายมิติ ทั้ง IP และ Port เพื่อความรวดเร็วในการ Audit ข้อมูลรายเครื่อง
- เพิ่มเติมให้มีการระบบ ตัวตนของ host และ subnet Match กับฝั่ง dest ที่เป็นวง 101

ข้อเสนอแนะเพิ่ม ในการประชุม:

- ควร lean ข้อมูลให้เรียบร้อยก่อน นำข้อมูลไป filter ipcdp ของ ISP ให้เรียบร้อยก่อนด้วย
- ในพาร์ทการ craft ACL ควรออกแบบด้วยรูปแบบ
	- Whitelist First, Explicit Deny Last 
	- group service port เป็นกลุ่ม ที่จะเข้าถึง server นั้นๆ
- Config ที่ผลิตออกมามีความสะอาด  พร้อมสำหรับการตรวจสอบ ( และพร้อมสำหรับการทำ Change Mgmt เพื่ออนุมัติก่อนนำไปประกาศใช้จริงบนอุปกรณ์ 
- ออกแบบ craft aclต่อไปและนำมาแสดงผลในที่ประชุม
- เตรียมแผนการทดสอบ (UAT) โดยการเปรียบเทียบ Blueprint กับนโยบายเดิม
- พิจารณาขยายผลการใช้งานไปยังวงเครือข่ายอื่นๆ ในเฟสถัดไป
