# 📄 บทสรุปอ้างอิง: ระเบียบวิธีและขอบเขตการจัดทำรายงาน (Data Refining Methodology)
> **เอกสารแนบท้ายสำหรับการตรวจสอบ Whitelist ACL (Subnet 10.27.101.0/24)**

รายงานฉบับนี้ถูกจัดทำขึ้นโดยดึงข้อมูลแบบเจาะจงและผ่านกระบวนการคัดกรอง (Data Refining) ตามมาตรฐานความมั่นคงปลอดภัย (Zero-Trust Architecture) เพื่อให้ได้ข้อมูลที่แม่นยำและสะท้อนถึงการให้บริการที่แท้จริงของเซิร์ฟเวอร์ โดยมีข้อกำหนดและคำนิยามในการจัดทำรายงานดังต่อไปนี้:

---

## 🔍 1. ขอบเขตของข้อมูลที่นำมาวิเคราะห์ (Data Scope)
ข้อมูลที่ปรากฏในรายงานคือ **"ทราฟฟิกขาเข้า (Inbound Traffic) ที่มาจากภายนอกวงเครือข่าย 101"** โดยมีเงื่อนไขดังนี้:
*   **✅ ข้อมูลที่ถูกรวมไว้:** ทราฟฟิกจากผู้ใช้งานวงอื่นๆ ภายในองค์กร (เช่น 10.27.120.x, 10.27.1.x) และทราฟฟิกจากอินเทอร์เน็ตภายนอก (Public IP) ที่พยายามเชื่อมต่อเข้ามายังเซิร์ฟเวอร์ในวง `10.27.101.x`
*   **❌ ข้อมูลที่ถูกตัดออก:** ทราฟฟิกการสื่อสารกันเองระหว่างเซิร์ฟเวอร์ภายในวง `10.27.101.x` ด้วยกันเอง (East-West Traffic) จะไม่ถูกนำมารวมในรายงานฉบับนี้ เนื่องจากวัตถุประสงค์หลักคือการทำ Firewall Policy เพื่อปกป้องขอบเขต (Perimeter) ด้านหน้าของระบบ

## 🛡️ 2. การคัดกรองพอร์ตชั่วคราวและผู้ให้บริการเนื้อหา (Ephemeral Port & CDN Filtering)
เพื่อให้ผู้ตรวจสอบสามารถมุ่งเน้นไปที่ **"พอร์ตมาตรฐาน"** ที่เซิร์ฟเวอร์ตั้งใจเปิดให้บริการจริงๆ ระบบได้ทำการคัดกรองข้อมูลที่เป็นเพียง **"ทราฟฟิกขากลับ (Return Traffic)"** ออกไปทั้งหมด โดยอ้างอิงจากหลักการต่อไปนี้:

*   **🔹 Ephemeral Ports (พอร์ตชั่วคราว):** 
    *   **คำนิยาม:** โดยปกติเมื่อเซิร์ฟเวอร์ของเรา (Client) ออกไปร้องขอข้อมูลจากอินเทอร์เน็ต (เช่น โหลดอัปเดตระบบ) ระบบปฏิบัติการจะสุ่มเปิดพอร์ตตัวเลขสูงๆ (โดยทั่วไปคือ พอร์ตที่มากกว่า 1024 ขึ้นไป เช่น 50000+) เพื่อรอรับข้อมูลกลับมา
    *   **เหตุผลในการตัดออก:** ข้อมูลที่วิ่งกลับมาหาพอร์ตเหล่านี้ไม่ใช่ผู้ใช้งานที่เข้ามาขอใช้บริการ แต่เป็นข้อมูลที่เซิร์ฟเวอร์ของเราร้องขอไปเอง ซึ่ง Firewall สมัยใหม่ (Stateful Firewall) จะทำการอนุญาตทราฟฟิกขากลับเหล่านี้ให้อัตโนมัติ (Established/Related) การนำพอร์ตชั่วคราวเหล่านี้มาเขียนเปิดไว้ถาวรใน Whitelist จะถือเป็นการสร้างช่องโหว่ร้ายแรงด้านความปลอดภัย
    *   **ข้อยกเว้น:** พอร์ตที่มากกว่า 1024 บางพอร์ตที่เป็น "มาตรฐานขององค์กร" (เช่น 3389 RDP, 8080 Web, 3306 MySQL) จะถูกคงไว้ในรายงานเพื่อนำมาพิจารณาตามปกติ
*   **🔹 CDN IPs (ผู้ให้บริการเครือข่ายส่งมอบเนื้อหา):**
    *   **คำนิยาม:** กลุ่มหมายเลข IP สาธารณะของผู้ให้บริการขนาดใหญ่ เช่น Cloudflare, AWS หรือ Facebook ซึ่งมักจะปรากฏคู่กับ Ephemeral Ports ในฐานะผู้ส่งข้อมูลกลับมา
    *   **เหตุผลในการตัดออก:** IP เหล่านี้ไม่ใช่ผู้ใช้งาน (End User) ที่มีจุดประสงค์เข้ามาใช้งานเซิร์ฟเวอร์ของเรา การตัดกลุ่มข้อมูลนี้ออกไป (ตามลอจิกของการตัดพอร์ตขากลับ) จะช่วยคัดกรองทราฟฟิกขยะ (Noise) ออกไป ทำให้ตัวเลข **Unique Sources Count (จำนวนผู้เข้าใช้งานที่ไม่ซ้ำกัน)** สะท้อนถึงกลุ่มผู้ใช้งานที่เป็นมนุษย์หรือระบบที่เจาะจงได้อย่างถูกต้องที่สุด
